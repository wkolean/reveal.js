Vagrant + Docker

Why?

  • Easy Onboarding
  • Easy development
  • Consistent environment

Why Vagrant

  • Text files instead of VM images
  • VMs customized per user
  • File sharing
  • Port forwarding
  • Free ($80 for VMWare)

Why Docker

  • Learn
  • Compartmentalize and isolate tech stack - create "Containers"
  • One process per container

Sounds stupid

But it's not

  • Shares kernel
  • Shares filesystem
  • Portable

Why Vagrant + Docker

  • Docker doesn't run natively on Windows/Mac
  • Vagrant makes it easy to create a local host (VirtualBox) that matches production
  • Vagrant (sometimes) makes Docker easier to work with
